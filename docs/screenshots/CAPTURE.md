# Screenshots — capture checklist

Drop the PNGs in this folder with the exact names below, then paste the README
block (bottom of this file) into `README.md` under a `## Screenshots` heading.

Shoot at phone width (~390px) for the deck shots — the UI is phone-first.

| File | What to capture |
|---|---|
| `home-sample.png` | The homepage "Sample analysis" stacked-card frame (truemirror.paahulhq.com). |
| `deck-cover.png` | Card 1 — the dark cover: Recovery/Sleep/Strain rings + stress + the verdict line. |
| `deck-attention.png` | The amber "What needs attention" card showing a live metric chip (e.g. HRV 44 ms ↓14). |
| `deck-action.png` | A "This week · 1 of 3" action flashcard. |

Tips:
- iPhone: open `/history?token=…`, swipe to the card you want, screenshot (side + volume-up).
- Trim status bars if you like, or leave them — they read as "real on a phone."
- Keep the three deck shots the same width so the README row lines up.
