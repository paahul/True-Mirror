# True Mirror — Build Plan

## Overview

iOS Shortcuts reads Apple Health + Fitness data from HealthKit, POSTs it to a Next.js API route, Claude analyses it and returns honest insights, results stored per user in Supabase (opt-in). Users install by tapping a shared Shortcut link posted to a group — no individual links, no manual provisioning. No App Store. No XML export.

---

## Stack

- **Next.js on Vercel** — API routes + history UI + shareable report pages
- **Supabase** — users + reports tables
- **Claude API** — analysis
- **iOS Shortcuts** — HealthKit data pipeline, shareable via icloud.com/shortcuts link

---

## How distribution works

You post one Shortcut link to a group (iMessage, WhatsApp, wherever). Anyone who taps it installs the Shortcut. On first run the Shortcut asks for their name, registers them via `/api/register`, and stores their token locally. Every subsequent run is silent — tap, wait 10 seconds, read the result.

Access is gated by an invite code embedded in the Shortcut. You set one code per group. Anyone with the Shortcut can register; anyone without it can't. To invite a second group, generate a new code and share a new Shortcut.

## Phase 1 — Backend

### 1a. Supabase schema

**users**
```
id         uuid primary key default gen_random_uuid()
name       text not null                          -- provided at registration
token      text unique not null default uuid      -- stored in Shortcut
email      text                                   -- optional, captured at registration
opt_in     boolean default true                   -- history opt-in
created_at timestamptz default now()
```

**reports**
```
id         uuid primary key default gen_random_uuid()
user_id    uuid references users(id)
raw_data   jsonb         -- full health payload as received
analysis   text          -- Claude's response
created_at timestamptz default now()
```

### 1b. POST /api/register

Self-serve registration — called once per user on Shortcut first run.

1. Validate invite code against `INVITE_CODE` env var (if set)
2. Accept `{ name, email?, invite_code? }`
3. Insert user row (token generated by DB default)
4. Return `{ token }`

Shortcut stores the token to iCloud Drive (`/Shortcuts/TrueMirror/token.txt`) and uses it on every subsequent run.

### 1c. POST /api/analyze?token=...

1. Look up user by token — return 401 if not found
2. Parse health JSON from body
3. Call Claude with health summary (see 1d)
4. If `user.opt_in`, save raw data + analysis to reports
5. Return `{ token, analysis, report_id, share_url, created_at }`

`share_url` is `https://truemirror.paahulhq.com/report/[report_id]` — returned so the Shortcut can offer a share action immediately after showing the result.

### 1d. Claude prompt design

Key principles:
- Honest, not relentlessly positive
- Lead with trends when present (HRV down 18% this week > HRV averages 42ms)
- Give 2-3 specific, actionable suggestions — not generic wellness advice
- Tone: smart direct friend, not a doctor, not a wellness app

Health summary passed to Claude includes trend indicators (↑/↓ % vs earlier in period) computed server-side from daily arrays. Claude is instructed to lead with trends when they're present.

### 1e. Health data shape

```json
{
  "period_days": 30,
  "steps":            { "daily": [...], "average": 7843, "total": 235290 },
  "sleep":            { "daily": [...], "average_hours_asleep": 6.4, "average_hours_in_bed": 7.8,
                        "avg_deep_minutes": 65, "avg_rem_minutes": 92, "avg_awake_minutes": 28 },
  "heart_rate":       { "resting_average": 68, "daily_resting": [...] },
  "hrv_ms":           { "average": 44, "daily": [...] },
  "vo2_max_ml_kg_min": 41.2,
  "active_energy":    { "daily": [...], "average": 412 },
  "exercise_minutes": { "daily": [...], "average": 22 },
  "workouts":         [{ "date": "...", "type": "Running", "duration_minutes": 42, "calories": 390 }],
  "stand_hours":      { "daily_average": 9 },
  "weight_kg":        { "latest": 82.4, "change_30d": -0.8 }
}
```

All fields optional. Claude skips metrics it doesn't have data for.

### 1f. GET /report/[id] — shareable report page

Public, read-only page. Anyone with the link can view. Security through obscurity — UUIDs are not guessable.

Shows:
- Date of analysis
- Full analysis text (formatted)
- "Get True Mirror" CTA at the bottom → referral loop

The CTA at the bottom is the referral mechanism. Someone shares their analysis to a group → people read it → tap "Get True Mirror" → install Shortcut → register. No attribution tracking needed for v1.

## Phase 2 — iOS Shortcut

### First-run flow (registration)

1. Try to read token from iCloud Drive (`/Shortcuts/TrueMirror/token.txt`)
2. If file doesn't exist:
   - Ask "What's your name?" (text input)
   - Optionally ask "Email for updates? (optional)" (text input, allow blank)
   - POST to `https://truemirror.paahulhq.com/api/register` with `{ name, email, invite_code: "[hardcoded]" }`
   - Save returned token to iCloud Drive file
3. Continue to analysis flow

### Analysis flow (every run)

1. Show "Fetching your health data..." notification
2. Pull each metric from HealthKit (see metric list below)
3. Build JSON payload
4. POST to `https://truemirror.paahulhq.com/api/analyze?token=[stored token]`
5. Show analysis via Quick Look or Show Result
6. Ask "Share this analysis?" → if yes, open share sheet with `share_url` from response

### Metrics to pull

| Metric | HealthKit action | Notes |
|---|---|---|
| Steps | Find Health Samples → Step Count | Sum per day, calculate average |
| Sleep | Find Health Samples → Sleep Analysis | Split by stage (Asleep, AsleepDeep, AsleepREM, Awake) — sum minutes per night per stage |
| Resting HR | Find Health Samples → Resting Heart Rate | One value per day |
| HRV | Find Health Samples → Heart Rate Variability | One value per day |
| VO2 max | Find Health Samples → VO2 Max | Most recent value |
| Active energy | Find Health Samples → Active Energy Burned | Sum per day |
| Exercise minutes | Find Health Samples → Exercise Time | Sum per day |
| Workouts | Find Workouts | Last 30 days |
| Weight | Find Health Samples → Body Mass | Most recent; first value for change_30d |
| Stand hours | Find Health Samples → Apple Stand Hour | Count `Stood` per day, average |

Sleep stages are the trickiest — iOS stores them as overlapping intervals, not pre-summed minutes. Calculate total minutes per stage in the Shortcut using Calculate Statistics on filtered samples.

HRV and VO2 max require Apple Watch. Handle gracefully when absent.

### Sharing

One Shortcut link posted to a group. Invite code hardcoded in the Shortcut. To invite a second group: duplicate the Shortcut, update the invite code, re-export, post the new link.

Long-term: Shortcut could show a "Invite a friend" action that surfaces the Shortcut link and invite code together.

## Phase 3 — History UI

Page at `/history?token=[token]`:
- Fetches reports for that user, newest first
- Shows date + first two lines of analysis as preview
- Tap to expand full analysis
- Share button on each report (generates `share_url`)
- Opt-in/out toggle → PATCH `/api/user/opt-in`

The Shortcut can open this page automatically after showing the analysis, or on demand via a menu action.

---

## Phase 4 — Prompt tuning

Tune with real data for:
- Sparse data (new Watch users, people who don't track sleep)
- Outlier weeks (illness, travel, unusual activity)
- Tone: honest without alarming, specific without medical
- Week-over-week comparison once history is populated (requires Phase 3)

Test harness: fixture files for different profiles (active, sedentary, poor sleep, great metrics) so prompt can be tested without running the Shortcut.

---

## Referral loop

The shareable report page is the loop:

1. User runs analysis, taps "Share" in Shortcut
2. Sends `/report/[id]` link to a friend or group
3. Friend reads the analysis, sees "Get True Mirror →" at the bottom
4. Taps → lands on truemirror.paahulhq.com
5. Installs Shortcut, registers

No attribution tracking needed at this scale. The landing page CTA is the referral mechanism.

Future: carry a `ref=[report_id]` param through to registration so you can see which analyses drive signups.
